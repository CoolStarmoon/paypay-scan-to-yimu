# 分类与导出体验优化需求文档

版本：沿用当前主需求版本，版本号不变

关联文档：`docs/category-mapping-requirements.md`

## 1. 背景

当前分类规则已经从硬编码分类扩展为“商户合集 + 用户分类映射”的机制。下一步需要继续优化分类覆盖范围、分类规则设置入口、导入导出层级，以及识别完成后的表格导出体验，让 App 更接近真实使用流程。

本需求作为当前分类映射需求的增量文档，不改变主需求版本号。

## 2. 目标

1. 调整商户合集结构，合并快餐和正餐，拆分电影和演出。
2. 扩充常见消费场景分类，覆盖实体店、线上购物、出行、居住缴费、公共服务、医疗等高频 PayPay 账单。
3. 优化分类规则设置入口和 UI，让设置更紧凑、更适合手机屏幕。
4. 优化分类配置导入导出流程，使其符合 Android 文件选择和保存习惯。
5. 优化识别完成后的表格导出方式，默认走 App 临时文件 + Android 分享接口，减少用户手动找文件的成本。
6. 增加“仅修正当前账单”能力，区分一次性修正和长期商户规则。

## 3. 分类规则调整

### 3.1 合并与拆分

原有分类调整如下：

| 调整项 | 旧商户合集 | 新商户合集 |
| --- | --- | --- |
| 合并 | 快餐、正餐 | 餐饮 |
| 拆分 | 电影娱乐 | 电影、演出 |

合并后，原 `restaurant_fast` 和 `restaurant_dining` 可以迁移到新的 `restaurant` 合集。拆分后，电影院、电影票、在线视频点播归入“电影”；演唱会、舞台剧、Live House、剧场、展览演出类票务归入“演出”。

### 3.3 兼容与迁移

已有用户配置需要兼容迁移：

1. 原快餐和正餐的用户分类映射迁移到“餐饮”。
2. 如果快餐和正餐原本映射不同，迁移时优先保留用户最近修改的映射，并提示用户可在分类规则设置中调整。
3. 原电影娱乐迁移时，电影相关关键词进入“电影”，演出、剧场、票务相关关键词进入“演出”。
4. 原有商户单独修正规则 `MerchantOverride` 不做自动改写，继续优先于商户合集规则。

## 4. UI 与功能调整

### 4.1 PayPay Icon 替换

导入截图区域上方当前显示的 PAY 图标，需要替换为用户提供的 PayPay 图标。

图标来源：

```text
E:/Downloads/cropped-logo_512_512.png
```

实现时应将该图片纳入 App 资源目录，不应运行时依赖下载目录路径。显示时保持原入口位置和尺寸约束，避免图标拉伸变形。

### 4.2 分类规则设置入口层级

分类规则设置不应直接出现在首页，而应从当前“商户合集”控件位置点进去设置。

推荐层级：

```text
商户合集
-> 分类规则设置
   -> 查看or编辑关键词规则
   -> 规则导入导出
```

要求：

- 分类规则设置进入二级菜单。
- 导入导出进入分类规则设置下的三级菜单。
- 分类规则设置入口需要在商户合集位置可见，用户能理解“先选合集，再设置该合集的分类”。

### 4.3 未设置提示弹窗

初次启动时，如果检测到用户尚未设置分类规则，应弹窗提示用户可以点击商户合集入口进行设置。

该提示不只在第一次启动时出现，而是每次启动都检测：

1. 如果用户尚未完成分类规则设置，则每次启动都弹窗提示。
2. 如果用户已完成至少一次有效分类规则设置，则不再弹出。
3. 弹窗应提供“去设置”和“稍后”两个操作。
4. 点击“去设置”后跳转到商户合集入口或分类规则设置入口。

### 4.4 分类规则设置 UI

分类规则设置 UI 需要更紧凑，减少不必要的技术字段展示。

要求：

- UI 中去掉英文 `group_id` 等技术字段，只显示中文合集名称。
- 分类输入框放在合集名称右侧，而不是下方。
- 一级分类和二级分类应在同一行或紧凑布局中展示。
- 二级分类允许为空。
- 长中文合集名称需要正常省略或换行，不能挤压输入框到不可用。

推荐行布局：

```text
便利店    一级分类 [食品餐饮]    二级分类 [便利店]
超市      一级分类 [食品餐饮]    二级分类 [食材采购]
餐饮      一级分类 [食品餐饮]    二级分类 [外出吃饭]
```

### 4.5 导入规则文案

导入规则入口不应直接在 UI 上写文件名，例如不直接显示 `category_mapping.csv`、`merchant_override.csv`、`rule_groups.csv`。

应使用用户能理解的说明文案：

| 导入项 | UI 文案 |
| --- | --- |
| 分类映射 | 导入账本分类映射 |
| 商户规则 | 导入商户识别规则 |
| 商户单独修正 | 导入单个商户修正规则 |

文件名和字段说明可以放在帮助文档、导入说明弹窗或错误详情中。

### 4.6 分类配置导出

点击“导出分类配置”时，应调用 Android 文件保存接口，让用户选择导出路径。

要求：

1. 调用 Android 系统文件选择/保存窗口。
2. 用户选择路径后再执行导出。
3. 导出成功后提示“已导出至 xx 路径”。
4. 用户取消路径选择时，不生成文件，并提示“已取消导出”或保持无打扰。
5. 导出文件使用 UTF-8 编码，中文和日文不乱码。

## 5. 识别完成后的表格导出流程

### 5.1 默认导出方式

识别完成后，不再直接导出到系统 Download 文件夹。

新的默认流程：

1. App 将一木记账导入表格生成到 App 自己的临时文件夹。
2. 生成成功后直接调用 Android 文件分享接口。
3. 弹出提示，用户可以点击“分享至一木记账 App”快速导入。
4. 下一次生成表格时，直接覆盖上一份临时表格，相当于清除缓存。

临时文件只用于当前导入流程，不作为用户长期保存文件。

### 5.2 自定义导出表格

原“重新导出表格”文案改为“自定义导出表格”。

点击后：

1. 调用 Android 文件保存接口。
2. 用户可以自行选择导出路径。
3. 导出的文件不属于临时缓存，不会在下一次生成表格时被自动覆盖或清除。
4. 导出成功后提示“已导出至 xx 路径”。

### 5.3 分享提示

默认导出完成后的提示需要明确告诉用户下一步：

```text
表格已生成，可以直接分享至一木记账 App 快速导入。
```

主要操作按钮：

```text
分享至一木记账 App
```

次要操作按钮：

```text
自定义导出表格
```

如果系统没有可用的一木记账分享目标，则仍打开系统分享面板，并允许用户保存或转发文件。

## 6. 修正分类功能

识别结果中修改某条账单分类时，原操作只有“取消”和“保存为商户规则”。需要新增一个中间选项：

```text
取消
仅修正当前账单
保存为商户规则
```

行为定义：

| 操作 | 行为 |
| --- | --- |
| 取消 | 不保存任何修改 |
| 仅修正当前账单 | 只修改当前识别结果中的这一条账单，不写入 `MerchantOverride` |
| 保存为商户规则 | 修改当前账单，并保存为该商户的长期修正规则 |

“仅修正当前账单”适用于同一商户偶尔发生不同用途消费的情况，例如便利店买文具、超市买日用品、酒店内餐厅消费等。

## 7. 验收标准

1. 快餐和正餐已合并为“餐饮”，电影和演出已拆分。
2. 新增商户合集覆盖线上购物、快递、手续费、外卖、打车、停车费、加油站、新干线、机票、酒店、景区门票、娱乐场所、房租、话费宽带、水电费、燃气费、公共服务、教务缴费、医疗，以及除服装店外的多类实体店。
3. 每个新增或调整后的商户合集至少包含 100 个商户名匹配关键词。
4. 导入截图区域上方图标已替换为用户提供的 PayPay 图标。
5. 未完成分类规则设置时，每次启动都会弹窗提示；完成设置后不再提示。
6. 分类规则设置从商户合集位置进入二级菜单，导入导出位于分类规则设置下的三级菜单。
7. 分类规则设置 UI 只展示中文合集名称，不展示英文技术字段，分类输入框位于合集名称右侧。
8. 导入规则 UI 使用说明文案，不直接展示文件名。
9. 导出分类配置会调用 Android 路径选择窗口，并在成功后提示“已导出至 xx 路径”。
10. 识别完成后默认导出到 App 临时文件夹，并调用 Android 分享接口，不再直接写入 Download 文件夹。
11. 默认临时表格会在下一次生成时覆盖；“自定义导出表格”生成的文件不会被缓存清理覆盖。
12. 修正分类弹窗包含“取消”“仅修正当前账单”“保存为商户规则”三个操作，并且三者行为符合定义。

## 8. 待确认问题

1. “分享至一木记账 App”是否能通过 Android Intent 精确定位目标 App，还是只能打开系统分享面板让用户选择。
2. 分类规则设置中是否允许批量编辑所有商户合集，还是必须逐个从商户合集详情进入。
3. 导入商户识别规则时，覆盖模式是否需要第一版开放，还是第一版只支持追加模式。
4. 迁移快餐和正餐映射冲突时，是否采用“最近修改优先”，还是弹窗让用户选择。
